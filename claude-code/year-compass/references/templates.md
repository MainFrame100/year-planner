## Правила заполнения

### Принцип авторства

> **ВСЕ записи в документе должны исходить от пользователя.**
> Ассистент НЕ формулирует выводы, факты, цели или уроки.

### Процесс верификации

При записи каждого пункта ассистент должен убедиться:

```
┌─────────────────────────────────────────────────────────────┐
│                    ЧЕКЛИСТ ВЕРИФИКАЦИИ                       │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  □ Это сформулировал пользователь своими словами?           │
│  □ Это проверяемый факт, а не предположение?                │
│  □ Достаточно ли конкретно? (есть детали, примеры)          │
│  □ Пользователь подтвердил, что это корректно?              │
│                                                              │
│  Если источник — предыдущие материалы:                      │
│  □ Актуально ли это сейчас?                                 │
│  □ Относится ли к данному разделу?                          │
│  □ Полностью ли отвечает на вопрос?                         │
│                                                              │
└─────────────────────────────────────────────────────────────┘
```

### Шаблоны уточняющих вопросов

**При поверхностном ответе:**
- «Можете привести конкретный пример?»
- «Что именно вы имеете в виду?»
- «Какие детали приходят на ум?»

**При использовании существующей информации:**
- «Вы упоминали [X] ранее. Это относится к этому разделу?»
- «Это всё ещё актуально или что-то изменилось?»
- «Полностью ли это отвечает на вопрос или есть что добавить?»

**При длинных ответах (предложение сократить):**
- «Это подробный ответ. Хотите, я предложу более лаконичную формулировку с сохранением ключевых деталей?»
- После предложения: «Это корректно отражает вашу мысль? Что нужно изменить?»
