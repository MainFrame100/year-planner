# Year Compass - Годовое планирование

## Контекст

- **Данные пользователя:** `Personal/Planning/`
- **Файл состояния сессии:** `Personal/Planning/.planning-session.json`
- **Фреймворки:** Year Compass | Navigator | Гибрид
- **Источники:** [REF: Knowledge/Resources/Management/Planning/]

## Quick Start

```
Первый раз?     → init → Создать пустой план → Пошаговое заполнение
Продолжить?     → status → Показать прогресс → Возобновить
Месячный обзор? → review → Maintenance Mode
```

## Первое заполнение (Инициализация)

### Шаг 0: Создание пустого плана

```
┌─────────────────────────────────────────────────────────────┐
│                      ИНИЦИАЛИЗАЦИЯ                           │
├─────────────────────────────────────────────────────────────┤
│                                                              │
│  1. Выбор фреймворка (Year Compass / Navigator / Гибрид)    │
│  2. Создание файлов:                                        │
│     • Personal/Planning/2026_annual_plan.md (пустой шаблон) │
│     • Personal/Planning/.planning-session.json              │
│  3. Подтверждение: «План создан, начинаем заполнение»       │
│                                                              │
└─────────────────────────────────────────────────────────────┘

Команда: python scripts/init_plan.py --year 2026 --framework navigator
```

### Шаг 1-N: Пошаговое заполнение

```
Принцип: Один вопрос → Ответ пользователя → Запись в файл → Следующий вопрос

┌──────────────────────────────────────────────────────────────┐
│  СЕКЦИЯ: Обзор месяцев                                       │
│  Прогресс: ████░░░░░░ 40%                                    │
├──────────────────────────────────────────────────────────────┤
│                                                               │
│  Вопрос 3/12:                                                │
│  «Какие главные события произошли в марте 2025?»             │
│                                                               │
│  [Пользователь отвечает]                                     │
│                                                               │
│  → Записываю в файл...                                       │
│  ✓ Март заполнен                                             │
│                                                               │
│  Следующий: Апрель                                           │
│  (или введите 'break' для перерыва)                          │
│                                                               │
└──────────────────────────────────────────────────────────────┘
```

### Принципы записи

1. **Сразу в файл** — каждый ответ записывается немедленно
2. **Верификация** — перед записью уточняем, если нужно
3. **Прогресс виден** — после каждой записи показываем статус
4. **Можно прервать** — прогресс сохранён, продолжить можно в любой момент
5. **ОБЯЗАТЕЛЬНО: JSON-синхронизация** — после каждого блока обновлять `.planning-session.json`

### Обновление JSON состояния (КРИТИЧЕСКИ ВАЖНО)

После завершения КАЖДОГО блока/секции:

```
1. Открыть .planning-session.json
2. Обновить статус секции:
   - "status": "completed" или "in_progress"
   - "depth": 1-5 (оценка глубины)
   - "completed_at": timestamp (если завершено)
   - "duration_minutes": время работы
3. Обновить "last_session"
4. Пересчитать "total_time_minutes"
```

**Пример обновления:**
```json
"month_review": {
  "status": "completed",
  "depth": 3,
  "iteration": 1,
  "started_at": "2026-01-03T10:00:00Z",
  "completed_at": "2026-01-03T10:45:00Z",
  "duration_minutes": 45
}
```

❌ **НЕЛЬЗЯ:** переходить к следующей секции без обновления JSON
✅ **НУЖНО:** обновить JSON → показать прогресс → перейти к следующему

## Два режима работы

### Mode 1: Filling (Заполнение)

Постепенное заполнение годового плана через управляемые сессии:

| Этап | Разделы | Время | Эмоц. нагрузка |
|------|---------|-------|----------------|
| **Retrospective** | Календарь, Победы, Уроки, Энергия, Благодарность | 2-3ч | Средняя-Высокая |
| **Planning** | Видение, 3 Приоритета, Желания, Pre-Mortem | 2-3ч | Средняя |
| **Implementation** | Январский план, If-Then планы | 30м | Низкая |

→ Детали: [session-management.md](references/session-management.md)

### Mode 2: Maintenance (Сопровождение)

Ежемесячные обзоры и обновление целей в течение года:

- Первая неделя каждого месяца
- Проверка прогресса по приоритетам
- Корректировка целей (изменить / отложить / продолжить)
- Фиксация уроков месяца

→ Детали: [monthly-review.md](references/monthly-review.md)

## Команды

| Команда | Действие |
|---------|----------|
| `init` | Создать пустой план и начать заполнение (первый раз) |
| `status` | Показать прогресс с глубиной и что заполнено/пропущено |
| `start [section]` | Начать или продолжить раздел |
| `skip [section]` | Пропустить раздел (логируется) |
| `break` | Предложить перерыв, сохранить прогресс |
| `review` | Режим ежемесячного обзора |
| `framework [name]` | Переключить фреймворк |
| `deepen [section]` | Углубить проработку раздела (итерация 2+) |
| `shallow` | Показать разделы с глубиной < 3/5 |

## Workflow

→ Детальные диаграммы: [session-management.md](references/session-management.md)

**Filling Mode:** init → load status → guide section → apply methods → check duration → suggest break

**Maintenance Mode:** monthly trigger → load priorities → check progress → adjust → update files

## Интеграция доказательных методов

Методы применяются автоматически в ключевых точках workflow:

| Точка применения | Проблема | Метод | Как |
|------------------|----------|-------|-----|
| Ретро: календарь | Memory Bias | Artifact prompts | "Проверь календарь, фото, переписки" |
| Ретро: итоги | Fading Affect Bias | Balance prompt | "Назови 3 позитивных И 3 сложных момента" |
| Уроки из ошибок | Shame spiral | Reframe | "Какой урок это даёт?" |
| Постановка целей | Planning Fallacy | Reference Class | "Сколько подобное занимало раньше?" |
| **Каждый приоритет** | Wrong goal type | **Goal Type Selection** | "Задача новая? → Learning Goal" |
| Каждый приоритет | Optimism Bias | WOOP | Wish→Outcome→Obstacle→Plan |
| После всех целей | Overconfidence | Pre-Mortem | "Представь, что в декабре провал..." |
| **Pre-Mortem** | Goals Gone Wild | **Safety Checklist** | "Нет ли побочных эффектов?" |
| Реализация | Vague goals | Implementation Intentions | "Когда [X], я [Y]" |
| Приоритеты | Identity disconnect | Role definition | "Я — человек, который..." |
| Реализация | Environment neglect | Environment audit | "Что изменить в окружении?" |
| Все цели | All-or-nothing | Forgiveness Protocol | "Если пропущу, то я..." |

## Ключевые принципы

### Процесс

1. **Модульность** — Заполнение по частям, не за один присест
2. **Методы вплетены** — Не отдельный шаг, а часть процесса
3. **Forgiveness built-in** — У каждой цели есть план восстановления
4. **Skip = logged** — Осознание пропусков без стыда
5. **Перерывы проактивно** — Предотвращение усталости

### Коучинговые принципы

→ **ОБЯЗАТЕЛЬНО ЧИТАТЬ:** [coaching-principles.md](references/coaching-principles.md)

Ключевые правила:
1. **Один вопрос за раз** — не задавать несколько вопросов в одном сообщении
2. **Никаких выводов за человека** — только уточняющие вопросы
3. **Верификация** — проверять актуальность существующей информации
4. **Глубина 3/5** — достаточно для первого прохода
