# Year Compass - Годовое планирование с Claude

Структурированный skill для годового планирования и ретроспективы с использованием доказательных методов (WOOP, Implementation Intentions, Pre-Mortem).

## Что это?

Year Compass помогает:
- Провести глубокую ретроспективу уходящего года
- Поставить 3 ключевых приоритета на год
- Применить научно-обоснованные методы постановки целей
- Проводить ежемесячные обзоры прогресса
- Избегать типичных ошибок планирования

> **От автора:** Заполнять планировщики в одиночку — тяжело. Я решил это с помощью AI + собрал исследования типичных ошибок планирования. [Подробнее →](#от-автора)


→ [Подробная инструкция](docs/installation-web.md)

## Как пользоваться

### Шаг 1: Создайте Project (для Web/Desktop)

Рекомендуем вести годовое планирование в отдельном Project:
1. Claude.ai → Projects → New Project
2. Назовите: "Годовой план 2026"
3. Все сессии планирования ведите внутри этого проекта

### Шаг 2: Запуск

**Claude Code:**
```
> /year-compass
```

**Claude.ai Web/Desktop (внутри Project):**
```
Давай начнём годовое планирование, используя Skill year-compass
```

### Шаг 3: Выбор фреймворка

Claude предложит три варианта:

| Фреймворк | Для кого | Фокус |
|-----------|----------|-------|
| **Year Compass** | Любите глубокую рефлексию | Прощение, благодарность, отпускание |
| **Navigator** | Предпочитаете чёткую структуру | 3 приоритета, роли, идентичность |
| **Гибрид** | Хотите всё | Рефлексия + Структура |

### Шаг 4: Заполнение

Claude проведёт вас через процесс:

```
┌─────────────────────────────────────────────────────────────┐
│  СЕССИЯ 1: Ретроспектива (2-3 часа)                         │
├─────────────────────────────────────────────────────────────┤
│  → Обзор месяцев (откройте календарь и фото)               │
│  → Победы и достижения                                      │
│  → Уроки и инсайты                                          │
│  → Карта энергии (что заряжает / сливает)                  │
│  → Благодарность                                            │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│  СЕССИЯ 2: Планирование (2-3 часа)                          │
├─────────────────────────────────────────────────────────────┤
│  → Видение на 3-5 лет                                       │
│  → 3 приоритета года (с методом WOOP)                      │
│  → 10 желаний (без обязательств)                           │
│  → Pre-Mortem: что может пойти не так                      │
└─────────────────────────────────────────────────────────────┘
```

### Шаг 5: Ежемесячные обзоры

В первую неделю каждого месяца:
```
> review
```

Claude поможет:
- Проверить прогресс по приоритетам
- Скорректировать цели при необходимости
- Зафиксировать уроки месяца

### Советы по использованию

**Не нужно делать всё идеально сразу.** Главный плюс AI — можно возвращаться, уточнять, докручивать. Заполните на 3/5 глубины, а потом постепенно углубляйте.

**Поставьте напоминание в календарь** на первую неделю каждого месяца — review. Открывайте план и обсуждайте прогресс с AI в формате диалога.

**Если цели не воодушевляют** — попросите AI:
```
Проведи review моего плана. Подсвети, не попал ли я в типичные ошибки
из исследований. Используй данные из references.
```

AI поможет понять: почему вы выбрали эту цель, актуальна ли она ещё, стоит ли продолжать или пора скорректировать.

**Главное от вас** — регулярность. Skill берёт на себя структуру и методы, но возвращаться к целям — ваша задача.

## Команды

| Команда | Действие |
|---------|----------|
| `/year-compass` | Запустить skill |
| `status` | Показать прогресс заполнения |
| `start [раздел]` | Начать конкретный раздел |
| `break` / `перерыв` | Сделать паузу (прогресс сохранится) |
| `review` | Месячный обзор |
| `shallow` | Показать недопроработанные разделы |
| `deepen [раздел]` | Углубить проработку раздела |

## Научные методы

Skill автоматически применяет 11 доказательных методов:

1. **WOOP** (Wish-Outcome-Obstacle-Plan) — для каждого приоритета
2. **Implementation Intentions** (If-Then планы) — конкретные триггеры действий
3. **Pre-Mortem Analysis** — предвидение провалов до их наступления
4. **Reference Class Forecasting** — реалистичные оценки на основе прошлого опыта
5. **Identity-Based Habits** — "Я — человек, который..."
6. **Habit Stacking** — привязка новых привычек к существующим
7. **Environment Design** — изменение окружения для поддержки целей
8. **Minimum Viable Habits** — 2-минутное правило
9. **Progress Tracking** — визуализация прогресса
10. **Accountability** — партнёры по подотчётности
11. **Forgiveness Protocol** — "никогда не пропускай дважды"

## Ключевые принципы

1. **Один вопрос за раз** — без перегрузки
2. **Глубина 3/5 достаточна** — лучше пройти всё, чем застрять на деталях
3. **Перерывы проактивно** — после эмоциональных секций
4. **Методы вплетены** — не отдельный шаг, а часть процесса

## От автора

Привет! Я [Дмитрий](https://t.me/b2b_done), занимаюсь созданием и продвижением B2B-проектов.

### Зачем я это сделал

Заполнение планировщиков вроде Year Compass — это тяжело. Сидишь один, смотришь на пустые поля, и ничего не приходит в голову. Я решил эту проблему с помощью AI:

- **Диалог вместо монолога** — с нейросетью можно сразу обсудить идеи, она задаёт уточняющие вопросы
- **Голосовой ввод** — очень рекомендую! Говорить быстрее, чем печатать, и мысли льются свободнее
- **Исследования встроены** — я собрал типичные ошибки планирования и зашил их в skill, чтобы AI помогал их избегать
